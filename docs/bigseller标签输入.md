# BigSeller 标签输入文件格式说明

本文档说明 BigSeller 批量标记订单时使用的输入文件格式。

---

## 文件格式

### Excel 文件结构

| 列号 | 列名 | 说明 | 必填 |
|------|------|------|------|
| A | `*订单号/包裹号(必填)` | 订单号 (order_sn) | 是 |
| B | `*订单标记(必填)` | 标签名称 | 是 |

### 文件命名规则

- 平台标签汇总: `order_tags.xlsx`
- 多文件拆分: `order_tags_1.xlsx`, `order_tags_2.xlsx` ...
- 每文件最多 1000 条记录

## 标签定义

### Shopee 平台标签

| 标签名称 | 判断条件 | 说明 |
|----------|----------|------|
| `同单多件` | 同一订单中任意一款产品购买数量 >= 2 | 单次购买多件同一商品 |
| `高频复购` | 同一顾客在 2 小时内有多次下单记录，且购买了同款产品 (item_id 相同) | 短时间内重复购买 |
| `地址偏远` | 菲律宾: rating=0 + 地址含 Mindanao/Visayas + 金额>6000PHP | 偏远地区需确认配送 |
| `历史退货退款派送失败` | 该买家历史存在 status=cancel 且有 tracking_number 的订单 | 有不良交易记录 |
| `顾客税务要求` | 聊天记录中包含税务关键词（税、税务、发票、invoice、tax 等） | 顾客需要税务单据 |
| `pass` | 不满足上述任一条件 | 正常订单，可发货 |
| `低分不发` | 顾客评分 > 0 且 < 3 | 差评顾客，暂不发货 |

| `地址偏远` | 菲律宾: 地址含 Mindanao/Visayas + 金额>6000PHP | 偏远地区需确认配送 |

## 数据筛选条件

### 待标记订单筛选

```
status = 'To Ship' AND tracking_number 为空
```

即：待发货状态且尚未生成运单号的订单。

## 注意事项

1. 标签可叠加：如订单同时满足多个条件，会在 Excel 中出现多行
2. 多平台区分：不同平台使用不同的任务脚本执行，但输出格式统一
3. 文件编码：Excel 文件使用 UTF-8 编码
